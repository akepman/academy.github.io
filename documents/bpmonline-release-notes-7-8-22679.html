Базовый интерфейс и возможности системы

Мини-карточки

  • Реализовано уведомление об отсутствии прав доступа при добавлении и изменении данных через мини-карточку.

  • Исправлено формирование гиперссылки в мини-карточке, если для раздела настроено несколько страниц редактирования. Теперь при переходе по гиперссылке учитывается тип записи.

Вход в систему

  • Реализована возможность использовать сохраненный в браузере пароль для входа в приложение.

  • Исправлена ошибка, возникавшая в некоторых случаях при входе в приложение пользователей, у которых выбрана культура с лунным календарем, например, ar-SA (Arabic — Saudi Arabia).

Работа в системе

  • При отображении реестра учтена ситуация, когда из объекта была удалена колонка, выведенная для отображения в реестр.

  • Реализовано корректное отображение удаленных колонок в настройке реестра. Если из объекта была удалена колонка, то она отображается как удаленная на странице настройки колонок.

  • Устранено длительное отображение маски загрузки, возникавшее вследствие нескольких одновременных пользовательских запросов к системе и, соответственно, наложения двух и более масок.

  • Исправлено обновление времени напоминаний. Если время начала активности на странице записи было изменено, то время напоминания обновится сразу, а не после обновления страницы.

Фильтры

  • Реализована возможность расширенной фильтрации по вхождению пользователя в функциональную или организационную роль. Так, например, с помощью фильтра можно отобрать список продаж, ответственный по которым входит в организационную роль “Прямые продажи”, вместо того, чтобы составлять список сотрудников вручную.

Фильтрация по вхождению пользователя в организационную роль “Прямые продажи”

 

  • Доработано сохранение настроек расширенных фильтров в случае, если в ранее настроенный фильтр были внесены изменения, а после применения обновленного фильтра осуществлялся переход в другой раздел либо в отфильтрованных записях вносились изменения на деталях.

  • Улучшено быстродействие окна выбора справочного значения в фильтрах для случаев, когда окно было открыто повторно, а до этого не было выбрано ни одного значения.

Инструменты управления клиентской базой

Поиск дублей

  • Обновлен механизм дедупликации при сохранении записей. Теперь локальный поиск дублей в разделах [Контакты] и [Контрагенты] выполняется аналогично глобальному поиску дублей и учитывает все действующие правила.

  • Реализованы новые правила поиска дублей в разделе [Контакты]:

•по полю [Мобильный телефон];

•по полю [Рабочий телефон];

•по полю [Email];

•по полю [ФИО], без учета порядка указания фамилии, имени и отчества.

  • Реализовано новое правило поиска дублей в разделе [Контрагенты] — по полю [Название].

Пример нового правила поиска дублей в разделе [Контрагенты]

 

  • Реализовано новое правило поиска дублей в разделе [Лиды], согласно которому в список потенциальных дублей попадут лиды, у которых совпадают данные в поле [ФИО контакта], независимо от порядка указания фамилии, имени и отчества.

  • Реализована возможность настраивать список колонок в реестре найденных дублей.

Настройка списка колонок в реестре дублирующихся записей

 

  • Исправлена ошибка, из-за которой в некоторых случаях не выполнялся переход к списку дублирующихся записей из уведомления о найденных дублях.

Мобильное приложение

  • Реализована автоматическая синхронизация конфигурации приложения. При наличии конфигурационных обновлений приложение автоматически загружает и применяет их без дополнительного подтверждения пользователем.

  • Устранено отображение маски загрузки, которая появлялась на какое-то время при открытии раздела еще до того, как пользователь начал вводить поисковый запрос.

Интеграции

Импорт данных из Excel

  • При успешной загрузке данных из Excel сообщение о возможности просмотреть детальный отчет в журнале загрузок данных не будет отображаться.

  • Обновлены рекомендации по подготовке файла Excel для импорта данных. Дополнены тексты уведомлений при загрузке некорректного файла, так что при загрузке некорректного файла вы сможете быстро определить причины ошибки.

Интеграция с почтой

  • Добавлена возможность открытия гиперссылки из загруженного в систему email-сообщения по клику. Ранее из-за ограничений модуля просмотра содержимого email-сообщений ссылку можно было открыть только с помощью команды контекстного меню.

  • Если синхронизация с MS Exchange настроена не для всего почтового ящика, а для выбранных папок, то при удалении одной из папок на почтовом сервере загрузка в систему новых писем из других синхронизированных папок выполнится корректно.

  • Исправлена ошибка недоступности выбора шрифта и размера текста на странице редактирования email-сообщения.

  • Исправлено отображение форматирования при просмотре писем формата plain text в почтовом клиенте MS Exchange. Ранее при отображении письма пропускались элементы форматирования (пробелы, переносы строк, табуляция).

  • Исправлена логика действий [Ответить], [Ответить всем] при переходе между разными письмами в bpm'online. Действия [Ответить] и [Ответить всем] срабатывают для текущего письма, а не для первого открытого.

Интеграция с календарем MS Exchange

  • Оптимизирован процесс синхронизации с Exchange Calendar. Если статус приглашения участника ("Подтверждено", "Отклонено" и пр.) не был изменен, то при синхронизации запрос в базу данных не отправляется.

  • Оптимизирован процесс синхронизации встреч с Exchange Calendar. Теперь автор встречи не изменится после нескольких циклов синхронизации, выполненных другим участником встречи.

  • Внесена доработка, согласно которой при синхронизации встреч с Exchange Calendar из bpm'online в Exchange передаются только первые 255 символов заголовка. Это помогает избежать обрезания заголовка в bpm’online после обратной синхронизации.

Bpm’online sales

  • В мобильном приложении реализована возможность работы с функциональностью Field Sales в режиме online.

  • На странице проекта реализовано автозаполнение поля [Контакт] при связывании проекта с продажей, у которой в поле [Клиент] указан контакт.

  • На странице проекта исправлена фильтрация в поле [Продажа] для связи проекта с продажей, которая относится к контакту или контрагенту, указанному в данном проекте.

Bpm’online service

  • Усовершенствована логика обновления данных на детали [Объекты обслуживания] страницы сервисного договора. После редактирования данных связанного с сервисным договором контакта изменения отобразятся на детали [Объекты обслуживания] сразу после обновления страницы сервисного договора.

  • Доработана проверка на валидность email-адресов в кириллице или с длинными доменными именами при получении сообщения службой поддержки.

  • В разделе [Релизы] на детали [Команда] при установке быстрого фильтра реализовано заполнение колонки фильтра значением по умолчанию “Ответственный / Группа”.

  • В разделе [Релизы] на детали [Плановые сроки] при установке быстрого фильтра реализовано заполнение колонки, по которой будет осуществляться фильтрация, значением по умолчанию “Этап”.

  • В разделе [Сервисы] на детали [Сервисные договоры] при установке быстрого фильтра реализовано заполнение колонки, по которой будет осуществляться фильтрация, значением по умолчанию “Сервисный договор”.

  • В разделе [Конфигурации] на детали [Обращения] при установке быстрого фильтра реализовано заполнение колонки, по которой будет осуществляться фильтрация, значением по умолчанию “Обращение”.

Заполнение колонки быстрого фильтра значением по умолчанию

 

  • На странице обращения улучшена визуальная подача подсказки о необходимости настроить часовой пояс пользователя.

Подсказка о необходимости настроить часовой пояс пользователя

 

  • Обновлена страница обращения:

•Теперь поля [Категория] и [Сервис] не являются обязательными для заполнения.

•Для пользователей продукта Service Enterprise поле [Сервисный договор] на странице обращения также не является обязательным для заполнения.

  • Обновлена логика автоматического изменения значения в поле [Причина закрытия] на странице обращения. В частности, теперь поле можно заполнить вручную на любом этапе обработки обращения. Подробнее логика работы поля рассмотрена в документации при описании страницы обращения.

  • Обновлена логика автоматического изменения значения поля [Ответственный] на странице обращения. Так, при переводе обращения в состояние “В работе” (по кнопке быстрого изменения состояния или при помощи индикатора процесса), если поле [Ответственный] не заполнено, оно заполняется значением ФИО текущего пользователя. Детальнее логика работы поля рассмотрена в документации при описании страницы обращения.

  • Настроено администрирование по записям в группах раздела [Обращения]. В частности, при создании новой группы она по умолчанию будет отображаться только автору записи. У автора есть возможность предоставить доступ к группе другим пользователям системы.

  • В CRM-линейке bpm’online выполнена группировка системных настроек, которые отвечают за автоматическую отправку email-уведомлений по обращениям. Теперь такие системные настройки входят в группу “Отправка email-сообщений”.

  • Доработана логика заполнения плановых сроков рассмотрения обращения. Значения полей [Время реакции] и [Время разрешения] рассчитываются с учетом часового пояса, установленного в профиле пользователя.

  • Для обеспечения чистоты данных при обработке обращений реализована логика по блокировке поля [Дата закрытия] на странице обращения.

  • Выполнены доработки процесса уведомления контактов об изменении состояния обращения. При выполнении действия [Запуск процесса “Отправка email-сообщения контакту обращения (при смене состояния обращения)”] автоматические уведомления не будут формироваться в случае, если контактом-заявителем в обращении и ответственным по обработке такого обращения является один и тот же человек.

  • Усовершенствован блок оценки в письме-уведомлении о решении обращения. Теперь для оценки обращения пользователю достаточно нажать на ячейку, которая содержит оценку, а не на значение оценки, как было реализовано раньше.

Бизнес-процессы

  • Реализовано сохранение в профиль браузера положение области элементов процесса. Теперь, если пользователь развернул/свернул область элементов процесса, то при очередном открытии дизайнера процессов она останется в том же виде.

Сохранение области элементов процесса в свернутом виде

 

  • Реализована возможность поиска по колонкам в окне множественного выбора колонок.

  • Добавлена возможность быстро очистить значение поля с маппингом при помощи отдельной кнопки.

  • Доработан механизм валидации формул в дизайнере процессов:

•введенное выражение проверяется при сохранении формулы;

•при обнаружении ошибок пользователь получает информацию о них;

•выполняется проверка синтаксической корректности выражения;

•проверяется соответствие типа вычисляемого выражения типу параметра, в который записывается результат вычисления.

  • Для удобства настройки свойств элементов бизнес-процессов добавлена возможность поиска в окне формулы – по элементам и параметрам элементов.

Инструменты пользовательской настройки

  • Доработан механизм сохранения изменений в мастере разделов и мастере деталей. Если при сохранении изменений возникла ошибка компиляции, отобразится соответствующее уведомление и изменения не сохранятся.

  • При настройке итогов добавлена возможность указать, учитывать ли настроенные в разделе фильтры для каждой отдельной серии графика, или для всех серий.

  • В мастере разделов улучшено расположение элементов в окнах редактирования названий групп полей и вкладок.

  • Появилась возможность добавлять текстовые поля в группе полей [Связи] страницы активности. Это позволяет связывать активность с объектом, который находится на внешних ресурсах, например, с web-страницей. Ранее активность можно было связать только с другими объектами системы, поскольку все поля связей заполнялись из системных справочников.

Текстовое поле в группе полей [Связи] страницы активности

 

  • В дизайнере контента внесен ряд доработок для лучшего отображения email-сообщений на разных устройствах и в разных почтовых клиентах, в частности на Gmail, Apple Mail, Outlook, Yahoo, Yandex, Mail.ru:

•Структура email-сообщения формируется в виде таблицы — <table>, а не отдельных блоков — <div>;

•Вместо embeded-стилей максимально используются inline-стили;

•В почтовом клиенте MS Outlook корректно отображаются строки по высоте;

•В Apple Mail корректно отображается цвет ссылок.

Производительность

  • Увеличена скорость открытия раздела [Итоги] и представлений [Итоги] каждого из разделов в случае, если в них настроено большое количество вкладок с аналитическими данными.

Инструменты разработки

  • Из мини-карточки и страницы записи общая логика была вынесена в родительский объект BaseEntityPage.

  • При вызове метода save BaseEntityPage выполняются общие для мини-карточки и страницы записи методы:

•проверка прав на редактирование сущности saveCheckCanEditRight;

•валидация заполненных значений saveAsyncValidate;

•сохранение сущности saveEntityInChain.

  • В BaseEntityPage созданы виртуальные дополнительные методы:

•processCheckCanEditResponse — обрабатывает результат проверки прав на редактирование (формирует объект с результатом проверки);

•validateSaveEntityResponse — обрабатывает результат сохранения сущности (проверяет ответ сервера по сохранению сущности).

  • Логируемая информация об ошибках, возникающих при работе с SVN, дополнена описанием корневой причины (RootCause) ошибки.

  • Исправлена ошибка, возникающая при попытке открыть в конфигурации замещающую схему объекта “Активность”.

  • Исправлена ошибка фиксации ресурсов в хранилище при разработке конфигурации в файловой системе.

Конфигурирование мобильного приложения
