Это документация Creatio версии 7.12.0. Мы рекомендуем использовать новую версию документации.

Страница лида

В bpm’online лид может создаваться несколькими способами:

•добавляется вручную в разделе [Лиды];

•импортируется из файла Excel;

•создается автоматически при посещении посадочной страницы потенциальным клиентом.

Страница лида (Рис. 1) состоит из панели инструментов (1), профиля лида (2), профилей юр.лица (3) и физ.лица (4). Также на странице представлены индикатор стадий (5), панель действий (6) и вкладки (7).

Рис. 1 — Страница лида

 

Панель инструментов (1)

Панель инструментов содержит кнопки для выполнения различных действий с записью.

Профиль лида (2)

Основная информация о лиде.

Тип потребности

Тип потребности потенциального клиента, на основании которой зарегистрирован лид. Например, категория продуктов или услуг вашего банка, в которых заинтересован потенциальный клиент. Поле обязательно для заполнения, предварительно настраивается в справочнике [Типы потребностей].

Как зарегистрирован

Способ регистрации лида в bpm’online. Например, лид может быть добавлен в систему вручную, создан автоматически при посещении потенциальным клиентом посадочной страницы (лендинга) либо зарегистрирован в результате обращения клиента.

Дата создания

Дата и время создания записи лида. Поле заполняется автоматически и недоступно для редактирования.

Ответственный

Сотрудник, назначенный ответственным за взращивание лида.

Предиктивный рейтинг

Вероятность успешной конвертации лида в продажу по шкале от 1 до 100 баллов. Подробнее >>> 

 

Профили юр. лица (3) и физ. лица (4)

Под профилем с основной информацией о лиде размещены профили юр. лица и физ. лица. В профиле юр. лица отображаются поля с названием, адресом web-сайта, отраслью и категорией компании, чья потребность фиксируется. В профиле физ. лица отображается ФИО контактного лица, его должность и мобильный телефон.

Если при создании записи лида в системе были указаны все необходимые данные, они автоматически отобразятся в профилях на странице лида после квалификации. Также профили могут быть дополнены основными данными на этапе квалификации. Подробнее >>> 

Индикатор стадий (5)

Индикатор стадий дает возможность отслеживать и выполнять стадии процесса управления лидом. Подробнее >>>

Панель действий (6)

На панели действий отображаются подсказки по списку задач ответственного. Здесь можно планировать задачи, отправлять email-сообщения, писать посты в ленту и фиксировать результаты, не покидая страницу лида. Подробнее >>>

Вкладка [Исходные данные]

Регистрационные данные

Группа полей содержит информацию о физ. лице, от которого зафиксирована потребность.

ФИО физ. лица

Фамилия, имя, отчество физ. лица. Если поле заполнено, то через иконку  доступен поиск данных о физ. лице в Linkedin. Для поиска в Facebook укажите почту физ. лица в поле [Email].

Должность

Должность, занимаемая физ. лицом, например, “Руководитель отдела”, предварительно заполняется в справочнике “Должности”.

Мобильный телефон

Мобильный телефон физ. лица.

Email

Адрес электронной почты потенциального клиента.

Название юр. лица

Наименование банка — потенциального клиента. Если поле заполнено, то через иконку  доступен поиск данных о банке в Linkedin и в Google.

Количество сотрудников

Количество сотрудников в банке.

Страна

Местоположение потенциального клиента.

Web

Адрес веб-сайта банка.

Похожие лиды

Деталь содержит список лидов, которые совпадают по параметрам, настроенным в правилах поиска дублей лидов. Так, на детали [Похожие лиды] отобразится запись с одинаковыми ФИО физ. лица, email-адресом и типом потребности.

Вкладка [Данные о потребности]

На вкладке содержится уточняющая информация о потребностях потенциального клиента.

Зрелость потребности

Уровень готовности клиента к приобретению продукта, от “Предполагается интерес” до “Готов к продаже”. Ответственный может выбирать значение поля и дополнять его необходимыми комментариями в процессе работы с лидом. Также значение может изменяться автоматически при выполнении процесса управления лидом, например, если настроен соответствующий бизнес-процесс.

Комментарий

Дополнительный комментарий о потребности клиента в приобретении продукта.

Параметры

Деталь для фиксации перечня специфических параметров потребности, которые хранятся в справочнике [Параметры].

Продукты

Деталь предназначена для формирования списка продуктов, по которым зафиксирован интерес.

Вы можете добавить на деталь только те продукты, которые настроены для выбранного типа потребности.

На заметку

Настроить продукты для типа потребности можно по действию [Настроить типы потребностей], которое доступно в меню действий раздела [Лиды].

Вкладка [Привлечение лида]

На вкладке фиксируется информация о том, каким способом был привлечен лид.

 

Канал

Канал получения лида — тип ресурса, с которого получен лид, например, “Веб: социальные сети”.

Источник

Конкретный ресурс, благодаря которому получен лид, например, “Facebook”.

Как зарегистрирован

Способ регистрации лида, например, “Добавлен вручную”, “Создан автоматически”.

Сайт перехода

Сайт, с которого пользователь перешел на посадочную страницу, в результате чего в системе был зарегистрирован лид. Поле недоступно для редактирования и заполняется автоматически при получении лида с посадочной страницы.

Лендинг

Название лендинга, из которого лид был передан в bpm’online. Поле недоступно для редактирования и заполняется автоматически при получении лида с посадочной страницы.

Вкладка [Хронология]

На вкладке содержатся представленные в хронологическом порядке записи системы, связанные с лидом. Подробнее >>>

Рис. 2 — Записи на вкладке [Хронология]

 

Вкладка [Данные о сделке]

На вкладке фиксируется детальная информация о сделке, связанной с лидом.

Продажа

Продажа, с которой связан лид. Поле заполняется автоматически на стадии перевода лида в продажу.

Ответственный за продажу

Менеджер отдела продаж, назначенный ответственным за продажу по лиду. В этом поле отображаются контакты, для которых в системе созданы пользователи. Поле может быть заполнено вручную при создании лида или на стадии перевода лида в продажу.

Направление продажи

Подразделение вашего банка, которое занимается переводом лида в продажу. Поле может быть заполнено вручную на стадии перевода лида в продажу.

Дата следующей актуализации

Планируемая дата актуализации данных по сделке.

Дата и время встречи

Назначенная дата ближайшей встречи с клиентом. Поле может быть заполнено вручную на стадии перевода лида в продажу.

Дата принятия решения

Ожидаемое время принятия клиентом решения по продаже. Поле может быть заполнено вручную на стадии перевода лида в продажу.

Вкладка [История]

На вкладке содержатся записи системы, связанные с лидом.

Стадии

Стадии, которые проходил и на которой в данный момент находится лид. Данные на деталь добавляются автоматически при переводе лида по стадиям и недоступны для редактирования.

Активности

Задачи, которые связаны с лидом. Отображает информацию из раздела [Активности]. Связь активности с лидом осуществляется по полю [Лид] страницы активности.

Звонки

Список входящих и исходящих звонков по лиду. Отображает информацию из раздела [Звонки]. Связь звонка с лидом осуществляется по полю [Лид] страницы звонка.

Email

Список сообщений, которые связаны с лидом. Связь email-сообщения с лидом осуществляется по полю [Лид] страницы email-сообщения.

Вкладка [Файлы и примечания]

Дополнительная информация по лиду, а также файлы и ссылки на веб-ресурсы, касающиеся лида. Подробнее >>> 

Файлы и ссылки

Используется для хранения файлов и ссылок, которые характеризуют лид. Например, на деталь можно добавить документы, отражающие историю взаимодействия с лидом, или полезные ссылки.

Примечания

Деталь предназначена для хранения любой дополнительной текстовой информации о лиде. Вы можете редактировать и форматировать текст примечаний непосредственно на детали. При переходе на другую вкладку страницы лида информация на детали [Примечания] сохраняется.
