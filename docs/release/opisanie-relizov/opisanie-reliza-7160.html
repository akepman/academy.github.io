Описание релиза 7.16.0

Дата релиза
20.04.2020
PDF

Представляем обзор изменений и улучшений, реализованных в Creatio версии 7.16.0.

Для перехода на новую версию системы воспользуйтесь инструкцией по обновлению.

Marketing Creatio 

Дизайнер контента 

  • Появилась возможность редактировать HTML-контент в шаблонах email без редактирования кода, непосредственно в дизайнере контента. Обновленная функциональность элемента HTML позволяет отметить в коде элемента переменные (текст, цвет шрифта или изображения) и настроить пользовательские макросы для редактирования.

Редактирование HTML-контента в дизайнере контента

  • Реализована возможность применять пользовательские шрифты в шаблонах email-рассылок. Для добавления шрифта укажите его URL в справочнике Пользовательские шрифты дизайнера шаблонов и дополните семейство шрифтов в справочнике Набор шрифтов дизайнера шаблонов.

  • Добавлена возможность навигации в дизайнере контента с помощью клавиши Esc. Таким способом можно перемещаться по структуре шаблона, закрывать модальные и всплывающие окна или поля для ввода.

Sales Creatio 

Планирование 

  • Появилась возможность изменить последовательность отображения колонок План, Факт и Факт, %. Колонки можно перетащить с помощью мыши. 

  • Добавлена возможность изменить ширину области с иерархией данных.

Изменение ширины колонки с данными

Creatio Marketplace 

  • Появилось новое приложение Field module для выполнения базовых задач полевых сотрудников. Решение позволяет планировать визиты в основном приложении и на мобильном устройстве, в том числе в автоматическом режиме (цикловые визиты). Доступны основные действия визита: “Чек-ин”, “Показ презентации”, “Чек-аут”. Также есть возможность создавать собственный набор правил и действий для визитов. Используя приложение как базовый модуль, с помощью low-code платформы Creatio можно реализовать решения для автоматизации задач полевых сотрудников разных сфер. Приложение совместимо со всеми продуктами Creatio.

Базовый интерфейс и возможности системы 

  • Устранена ситуация, когда предупреждение о необходимости заполнить обязательное поле на странице продолжало отображаться даже после изменения условий обязательности в соответствии с настроенным для поля бизнес-правилом.

Предиктивный анализ данных 

  • Реализована возможность обучения рекомендательных моделей прогнозирования. На основании взаимодействия между объектами (например, совершенные ранее покупки или просмотр товара) система может рекомендовать продукты, которые клиент приобретет с наибольшей вероятностью, или такие, которые ранее клиентом не приобретались.

Добавление рекомендательной модели машинного обучения

  • Появилась возможность получать прогноз по заполнению полей записи сразу при ее создании, до сохранения в системе.

Аналитика 

  • Появилась возможность настраивать вертикальные и горизонтальные гистограммы “с накоплением” (stacked). Каждый столбец такой диаграммы содержит сумму значений входящих в него столбцов. Такие графики полезны, когда необходимо сравнить не только отдельные значения, но и суммарный показатель в целом. Например, на графике можно одновременно отобразить:

    • сумму продажи на каждой стадии по разным сотрудникам;

    • общую сумму продаж по сотруднику;

    • вклад каждой стадии в общую сумму.

      Для настройки выберите тип графика “Гистограмма” или “Горизонтальная гистограмма” и установите признак С накоплением.

Мобильное приложение 

  • В телефонии Asterisk появилась возможность указать, какой из двух протоколов — SIP или PJSIP — будет использоваться.

  • Появилась возможность безусловной (слепой) переадресации звонка для коннекторов Avaya, Webitel, Oktell, Finesse, TAPI, Infinity и Asterisk. Это упрощает процесс обработки звонка оператором.

  • Расширены возможности базовой интеграции с телефонией Avaya. Так, можно получить расширенную статистику по звонку. Для этого в раздел Звонки добавлен уникальный идентификатор звонка GloballyUniqueCallLinkageId.

Интеграции 

  • При добавлении новой или удалении существующей учетной записи почты изменения отображаются на коммуникационной панели сразу же, без необходимости обновления страницы.

  • При успешном добавлении учетной записи в Creatio справочник Домены почтовых провайдеров автоматически дополняется данными нового почтового провайдера. В результате при настройке других учетных записей почты система распознает провайдера по доменному имени и пользователям не придется указывать его вручную.

  • Оптимизирован процесс синхронизации почты пользователей Creatio, у которых настроена интеграция с почтовым провайдером по протоколу IMAP. Теперь отбор новых писем с сервера выполняется на стороне отдельного микросервиса, что позволяет избежать избыточного загромождения очереди планировщика и ухудшения общей производительности приложения. Улучшения реализованы на базе библиотеки MailKit с использованием стратегии UID.

Бизнес-процессы 

  • Появилась возможность конвертировать описательные процессы из Studio free в исполняемые процессы в другие продукты Creatio. После моделирования и согласования бизнес-процесса в Studio free экспортируйте его в файл формата *.bpmn, а затем импортируйте в нужный продукт Creatio для дополнительной настройки и автоматизации. Также файл можно перетащить в рабочую область дизайнера процессов. Так можно переносить процессы c любых платформ, которые поддерживают выгрузку в формат *.bpmn.

  • Появилась возможность остановить бизнес-процесс, в схеме которого отсутствуют прерывающие элементы, например, промежуточное событие Обработка таймера или элементы, предполагающие действия пользователя в ходе процесса. Ранее процесс, состоящий только из действий системы (например, элементы Добавить данные, Изменить данные   и   т.   д.), можно было остановить только после выполнения всех элементов.

  • Для удобства анализа диаграммы выполнения процесса реализована подсветка входящих и исходящих потоков элемента.

Подсветка потоков элемента на диаграмме выполнения процесса

Дизайнер процессов 

  • Стало проще ориентироваться в сложных процессах c большим количеством шагов. Специальная навигационная панель позволяет отобразить мини-карту диаграммы процесса, включая элементы, которые не помещаются в рабочее пространство дизайнера. Кроме мини-карты, на навигационной панели доступны разные варианты управления масштабом диаграммы. Также навигационная панель доступна на диаграмме выполнения бизнес-процесса в журнале процессов, на диаграмме зависимостей пакетов в разделе Управление конфигурацией и в дизайнере кампаний.

  • Для улучшения читабельности процесса появилась возможность с помощью мыши перемещать подписи элементов и потоков на диаграмме бизнес-процесса.

  • Непрерывающие стартовые события в событийных подпроцессах теперь отмечаются пунктирной границей в соответствии со стандартами нотации BPMN.

Инструменты пользовательской настройки 

  • Реализован новый интерфейс для настройки печатных форм. Настройка выполняется в разделе Настройка отчетов. Перейти в раздел можно из дизайнера системы или из справочника Печатные формы. При формировании отчета вы можете использовать подсказки по заполнению полей и деталей, а также инструкцию “Настройка печатных форм MS Word”.

Важно. В версии 7.16.0 функциональность настройки печатных форм в новом интерфейсе доступна в режиме бета-тестирования. Чтобы оценить новые возможности раздела, запросите включение функциональности в службе технической поддержки. Будем благодарны за обратную связь. В открытом доступе функциональность появится уже в ближайших релизах. Все отчеты, настроенные в старом интерфейсе, будут доступны для просмотра и изменения.

Дизайнер отчетов

Администрирование 

  • Для удобства распределения лицензий появилась возможность анализировать сводные данные по количеству доступных и использованных лицензий на специальной диаграмме.

Сводная информация о количестве лицензий

  • Если срок лицензий истек, то сразу после входа в систему пользователь получит уведомление об этом и будет переадресован на обновленную страницу распределения лицензий.

    На странице доступны следующие возможности:

    • сформировать запрос на новые лицензии;

    • загрузить новые лицензии;

    • выбрать необходимую лицензию и распределить между пользователями.

      Перейти в менеджер лицензий также можно из дизайнера системы, блок “Пользователи и администрирование”.

  • Реализована поддержка платформы .NET Core продуктами Service Enterprise, Sales Enterprise и Studio Enterprise. Подробнее о поддержке продуктами Creatio платформы .NET Core можно узнать в статье Продукты Creatio на платформе .NET Core.

Инструменты разработки 

  • В веб-службе Data Service реализован режим upsert при обновлении данных с помощью UpdateQuery. В этом режиме, если операция обновления не была выполнена успешно по причине отсутствия записи, то будет выполнена операция добавления записи. Это позволяет сократить в два раза количество запросов, если происходит полная синхронизация записей объектов.

    Для включения режима необходимо указать необязательный параметр запроса isUpsert и установить для него значение “true”. UpdateQuery в режиме upsert должен cодержать коллекцию значений всех колонок, которые нужны для добавления записи, а не только измененных на стороне источника данных, и использоваться только при обновлении одной записи.

  • Реализована возможность в коде отправить письмо через SMTP с постоянными параметрами подключения, без добавления ящика через интерфейс. Подробная информация доступна в статьях “Отправка email-сообщения с явным указанием учетных данных” и “Отправка email-сообщения с существующей учетной записи”.

  • Для протокола OData 4 реализована возможность работы с типом данных stream для передачи изображений, файлов и двоичных данных.

  • Появилась возможность создавать новые модули приложения с помощью фреймворка Angular. Больше информации доступно в статье “Файловый контент пакетов”.

  • На диаграмме зависимостей пакетов в разделе Управление конфигурацией добавлена подсветка связей по зависимостям с другими пакетами.
